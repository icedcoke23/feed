# 照片批量上传 + PDF就地编辑 设计文档

## 功能一：批量上传 + 可选裁剪

### 现状
- 裁剪流程禁用了 `multiple`，每次只能选一张照片再裁剪
- 原始上传流程支持多选但无裁剪功能
- 两个流程互斥（`useCropFlow` 判断）

### 改进
- 上传区域始终支持 `multiple` 多选，选择后直接上传（不弹裁剪框）
- 每张照片 hover 时操作按钮：裁剪 + 替换 + 删除
- 点击「裁剪」打开裁剪对话框，裁剪完替换该照片
- 上传按钮文案改为"点击或拖拽上传照片（支持多选）"

### 涉及文件
- `src/components/business/student-photos.tsx` — 移除 `useCropFlow` 分支，统一为直接上传+可选裁剪
- `src/hooks/use-feedback-form.ts` — `handlePhotoUpload` 保持批量逻辑，`handleAddCroppedPhoto`/`handleReplacePhoto` 用于裁剪后操作

### 交互
1. 用户点击上传/拖拽 → 多文件直接上传压缩 → 显示在照片网格
2. hover 照片 → 显示「裁剪」「替换」「删除」三个按钮
3. 点击「裁剪」→ 打开裁剪对话框 → 裁剪完替换原图
4. 点击「替换」→ 选择新文件 → 直接替换（不强制裁剪）

## 功能二：PDF预览页就地编辑

### 现状
- PDF页面只能"返回编辑"跳回 new 页面
- 无法直接在PDF预览中修改内容

### 改进
- 每个文本区域（学员优点、能力提升、需要提升、教师建议）右上角添加编辑按钮
- 点击后该区域变为可编辑 textarea，编辑完点击完成保存
- 照片区域添加「编辑照片」按钮，弹出照片管理弹窗
- 所有编辑直接更新 reportData 状态，PDF预览实时刷新

### 涉及文件
- `src/app/feedback/pdf/page.tsx` — 添加就地编辑功能

### 交互
- 非编辑模式：正常显示，hover时显示编辑图标
- 编辑模式：区域边框变蓝，显示textarea + 完成按钮
- 照片编辑：弹窗内可添加/删除/裁剪照片，关闭弹窗后更新预览
